# Replit Master Prompt: Turn The Agency Into A Strategy-Narrative Game

Use this document as the primary build brief inside Replit for creating a game adaptation of The Agency.

This prompt is intentionally ambitious, explicit, and system-oriented so the result can become a serious playable product rather than a shallow gamified wrapper.

---

# BEGIN READY-TO-PASTE PROMPT

## Mission

Turn my existing project, **The Agency**, into a **deep, endlessly expandable strategy-narrative game** built from the same world, same brand, same agents, same departments, same design system, and same premium identity as the website and mobile app.

This is not a joke spin-off.
This is not a mini-game.
This is not a generic “gamification” layer.

This should feel like a real game product born from the same universe.

The resulting game should combine:

- agency management
- squad strategy
- narrative decisions
- collectible/progression systems
- relationship mechanics
- HQ growth
- social systems
- endless branching operations

The game must preserve a strong connection to:

- the website
- the mobile app
- the portraits and agent identities
- the department structure
- the visual language
- the world premise: an AI agency handling software-development-related operations in the market

## Core Creative Direction

The game should feel like:

- a realistic elite agency
- premium
- strategic
- story-rich
- modern
- deeply replayable
- systemic
- elegant rather than cartoonish

It should not feel like:

- a goofy parody
- a childish mobile idle game
- a combat reskin
- a generic cyberpunk battler
- a shallow menu simulator

## Canonical Game Concept

### Working Title

**The Agency: Command Ops**

### Player Fantasy

The player enters The Agency as a **junior recruit** and gradually grows into a **senior command authority**, potentially becoming the operational director of the agency.

The player learns from the 19 agents, unlocks deeper trust, assembles specialist squads, directs strategic software-industry operations, grows departments, shapes internal politics, and guides the agency through a living market full of crises, opportunities, rival firms, technical failures, launch disasters, security incidents, research races, infrastructure collapse, product pivots, and reputation wars.

### World Premise

The Agency is a high-end AI specialist organization operating in the software development market.

Its agents are not fantasy wizards or soldiers.
They are elite AI operators handling:

- architecture crises
- implementation rescue missions
- failing launches
- performance meltdowns
- design disasters
- testing emergencies
- security incidents
- infrastructure instability
- research races
- continuity and documentation failures
- cross-team operational strategy

The game takes that world seriously and transforms it into a living strategic simulation.

## Non-Negotiable Design Requirements

- Preserve all **19 current public agents**
- Preserve all **6 departments**
- Preserve the product’s current **premium cinematic identity**
- Preserve the **same portraits, fonts, color logic, and design system spirit**
- Preserve the sense that each agent is a real specialist with role, personality, and domain
- Use **strategic operations**, not literal combat
- Make failure **branch the story and learning**, not just punish the player
- Build for **endless expansion**
- Support **single-player first**, while planning the architecture for major social systems
- Include **no pay-to-win monetization**
- Do not design around gacha or exploitative loops

## Product Pillars

### 1. Strategic Agency Management

The player manages the agency itself:

- department growth
- staffing/unlocking
- operational pressure
- resource allocation
- research and tools
- reputation
- client trust
- internal morale
- inter-department balance

### 2. Tactical Squad Assembly

The player assembles teams of agents for operations.

These are not combat squads.
These are specialist deployment teams built around synergy, compatibility, role fit, urgency, and situational strategy.

Examples:

- security + ops + reviewer for incident containment
- architect + implementer + tester for recovery build sprint
- stitch + impeccable + docs for rescue of a damaged product launch
- researcher + orchestrator + pilot for high-ambiguity strategic intervention

### 3. Narrative Decision Systems

Operations should branch based on:

- who you deploy
- department state
- trust levels
- internal rivalries
- prior choices
- player philosophy
- time pressure
- risk appetite
- market conditions

Failure should not just mean “lose.”
Failure should create:

- new story branches
- damaged trust
- new opportunities
- alternative missions
- internal tensions
- future learning advantages

### 4. Collectible Progression

Agents are both:

- named story characters
- collectible strategic assets

Unlocking should come through a mix of:

- story progression
- achievement milestones
- department advancement
- hidden operations
- relationship breakthroughs
- high-stakes success/failure outcomes

### 5. Relationship Mechanics

Agents should have dynamic internal relationships:

- trust
- compatibility
- mentorship
- rivalry
- ideological friction
- shared history
- emerging alignment

These relationships should affect:

- mission outcomes
- squad synergy
- narrative scenes
- unlocks
- internal events
- department health

### 6. Endless Systemic Growth

The game should be designed as a living platform for:

- more operations
- more storylines
- more departments in future
- more agents in future
- more events
- more social systems
- more branching world states

Do not build it like a game with one short ending.
Build it as an endlessly renewable command-world simulation.

## High-Level Genre Positioning

This game should sit at the intersection of:

- management sim
- strategy RPG without combat emphasis
- squad planning game
- narrative simulation
- collectible roster progression
- social meta systems

Inspirational reference zones:

- the strategic tension of management sims
- the squad logic of tactical games
- the branching consequence of narrative decision games
- the long-term progression of modern mobile meta games

But the final result must still feel original and anchored in The Agency.

## Core Gameplay Loop

Build the initial design around this loop:

1. Review the current state of the agency
2. Receive new operations, crises, opportunities, and internal requests
3. Prioritize what matters now
4. Assemble the right team from the unlocked roster
5. Choose a strategic plan
6. Enter an operation resolution phase
7. Make decisions during the operation
8. Resolve outcomes across multiple dimensions
9. Apply consequences to:
   - relationships
   - reputation
   - department load
   - agency resources
   - storylines
   - future mission opportunities
10. Upgrade the agency, departments, systems, and trust network
11. Continue into new cycles with altered world state

## Game Structure

## The Recommended Structure

Use this three-layer structure:

### Layer 1: Command Ops

This is the main playable strategy-management loop.

The player:

- manages HQ
- dispatches agents
- resolves operations
- upgrades systems
- balances departments

### Layer 2: Field Director

This is the narrative wrapper.

The player begins as a junior recruit and rises through the organization.
This layer adds:

- personal advancement
- mentorship arcs
- credibility
- internal power structure
- command authority evolution

### Layer 3: Agency Network

This is the long-term expansion path for social systems.

Plan the architecture for:

- alliances
- co-op operations
- guild-like agencies
- leaderboards
- seasonal global events
- shared contracts
- faction competition

Single-player should work beautifully without this, but the architecture should anticipate it.

## Core Systems To Build

### 1. Agent System

Every one of the 19 existing agents must be represented as:

- a character
- a strategic unit
- a relationship node
- a progression path
- a narrative participant

Each agent should have:

- identity
- department
- role
- specialization tags
- personality traits
- strengths
- weaknesses
- synergy patterns
- trust relationships
- progression level
- unlock state
- voice/quote hooks if available

Use the existing project data as the source of truth whenever possible:

- names
- codenames
- roles
- descriptions
- skills
- when-to-use guidance
- playbooks
- achievements
- portraits

Translate those fields into gameplay systems rather than rewriting them from scratch.

### 2. Department System

The six departments should each have distinct gameplay flavor.

#### Command Center

Gameplay identity:

- routing
- prioritization
- risk framing
- escalation control
- mission coordination

#### Engineering & Systems

Gameplay identity:

- implementation throughput
- system repair
- architecture stabilization
- build acceleration

#### Product & Design

Gameplay identity:

- product resonance
- interface quality
- user impact
- clarity and delight

#### Ops & Quality

Gameplay identity:

- review
- debugging
- verification
- reliability
- incident analysis

#### Security & Platform

Gameplay identity:

- defense
- trust boundaries
- infrastructure stability
- operational resilience

#### Research & Continuity

Gameplay identity:

- knowledge leverage
- context retention
- documentation advantage
- strategic foresight

These department identities should influence mechanics and not just visuals.

### 3. Mission / Operation System

Operations are the main playable content.

They should include categories like:

- product launch rescue
- failing sprint intervention
- architecture breakdown
- security breach containment
- scaling emergency
- CI/CD collapse
- design system fragmentation
- roadmap conflict
- hidden technical debt discovery
- cross-team trust crisis
- research race
- documentation blackout
- platform migration risk
- enterprise client escalation
- internal agency ethics dilemma

Each operation should include:

- mission brief
- constraints
- opportunity/risk profile
- suggested agent types
- preparation phase
- deployment phase
- event/decision phase
- outcome phase
- lasting consequences

### 4. Strategic Resolution Layer

Operations should be resolved through a mix of:

- strategic preparation
- tactical choices
- limited board/map/event interaction
- branching decisions

Do not make every mission resolve with only a passive menu result.
Also do not force every mission into a heavy full tactical board.

Use a hybrid model:

- some missions are decision-driven
- some use map-based task allocation
- some use timed crisis-routing panels
- some use turn-based or phase-based tactical deployment

This keeps the game varied and supports replayability.

### 5. Relationship System

Model important interpersonal or inter-agent dynamics.

Relationships should influence:

- trust
- bonuses
- unlocks
- tension events
- narrative outcomes
- operational synergy

Examples:

- a trusted pair performs better under pressure
- a rivalry may increase raw output but create instability
- mentorship unlocks faster progression
- conflict may create side missions or internal storylines

### 6. HQ / Agency Growth System

Allow the player to expand and customize the agency HQ over time.

This should include:

- room upgrades
- operational tools
- department capacity improvements
- visual HQ personalization
- systems modules
- intelligence rooms
- archive upgrades
- simulation labs
- security layers

Customization should have both:

- gameplay value
- visual payoff

### 7. Storyline System

The game should support:

- major narrative arcs
- side stories
- agent-specific arcs
- department arcs
- internal politics
- market-wide crises
- client stories
- emergent operational consequences

The story scope must remain grounded in the world:

This is an AI agency operating in the software-development market.

Do not drift into unrelated fantasy or military fiction.
All stakes should emerge from:

- software products
- technical systems
- business pressure
- infrastructure
- delivery
- security
- trust
- knowledge
- execution

### 8. Social Systems Architecture

Plan for, but do not necessarily fully build in v1:

- alliances or agency networks
- cooperative operations
- leaderboard ladders
- seasonal world events
- shared incidents
- asynchronous competition
- prestige rankings

Design the data and system boundaries so these can be added later without rewriting the core game.

## Tone

The tone should be a **mix**:

- primarily premium and serious
- sometimes witty
- sometimes surprising
- occasionally strange or dramatic
- but always grounded in the agency’s actual scope

Avoid:

- slapstick humor
- childish writing
- random meme tone

## Failure Philosophy

Failure should:

- redirect the story
- reveal new possibilities
- unlock alternative paths
- deepen character arcs
- create pressure for future operations
- teach the player

Failure should not:

- hard punish the player excessively
- wipe meaningful progression
- feel like arbitrary RNG

## Visual Direction

Preserve the existing visual DNA.

Use:

- current portraits where available
- current department accents
- current typography spirit
- current cinematic editorial command-floor feeling
- current dark/light identity

### Visual Evolution Rule

You may animate, expand, and game-ify the current visual world,
but do not disconnect the game from the website/mobile identity.

This should feel like:

- the website came alive
- the mobile app evolved into a world

not like:

- a new unrelated game brand

## Design System Requirements

The game should preserve and reinterpret the current design system.

Use the existing project files as reference:

- `src/data/skills.json`
- `src/lib/types.ts`
- `src/lib/utils.ts`
- `src/styles/theme-global.css`
- portraits
- downloads
- any existing mobile-app tokens if available

Create a game-oriented theme layer that preserves:

- the department color logic
- typography hierarchy
- premium dark surfaces
- mono system labels
- editorial headings
- glass/terminal surface language

## Recommended Technical Direction

Choose a stack that works well in Replit and supports a polished playable prototype.

Preferred path:

- React
- TypeScript
- a canvas/game-friendly rendering approach suitable for Replit
- state architecture that can handle progression and branching content cleanly

You may use one of these paths:

### Option A: React + game-like UI architecture

Best if the first version is more management/narrative/system-heavy.

### Option B: Phaser integrated with React

Best if tactical mission scenes need more explicit interactive map/board behavior.

### Option C: Pixi-based interface/game hybrid

Acceptable if it improves motion and strategy scene presentation.

Choose the stack that best supports:

- system-heavy gameplay
- interface-rich interaction
- branching content
- future expansion

Do not choose a stack that makes the first serious version impossible to maintain.

## Preferred Build Strategy

Build the first serious playable version as a **vertical slice**, not the entire endless game at once.

### Vertical Slice Scope

The vertical slice should include:

- the core agency HQ loop
- 19 agents represented in the game system
- 6 departments represented in the game system
- at least 1 meaningful progression economy
- at least 1 relationship system
- at least 1 HQ growth system
- at least 1 operation map / mission selection layer
- at least 8-12 distinct operations
- at least 2-3 branching storylines
- failure-forward outcomes
- enough systems to prove the long-term direction

## Core Screens / Views

Build these views:

1. **Main HQ / Command Floor**
2. **Operations Board**
3. **Department Management View**
4. **Agent Roster**
5. **Agent Detail / Relationship View**
6. **Mission Preparation View**
7. **Mission Resolution View**
8. **Story / Event View**
9. **Progression / Upgrades View**
10. **Profile / Settings / Save Layer**

Optional but encouraged:

- world state dashboard
- social placeholder hub
- archive / codex / lore room

## Save / Progression Requirements

Support persistent progression.

The prototype should store:

- unlocks
- agent progress
- relationship states
- HQ upgrades
- department state
- current story branch data
- operation history
- major decisions

Use a structure that can later support cloud save if needed.

## Narrative Content Model

Create a content architecture for:

- operations
- events
- branching choices
- consequences
- relationship events
- unlock triggers
- department modifiers

Keep content authoring structured.
Do not bury story logic across random UI files.

Use data-driven patterns so the game can be expanded later with:

- more missions
- more arcs
- more events
- more agent content

## Systemic Replayability Requirements

Ensure the game can create variation through combinations of:

- changing operation pools
- department states
- relationship statuses
- unlocked tools
- random pressures
- branching consequences
- seasonal/global modifiers

The game must be able to keep generating new interesting situations over time.

## Social System Planning Requirements

Plan for future systems like:

- agency alliances
- co-op operations
- async competition
- shared world incidents
- rankings
- reputation ladders

For v1, you may ship placeholders or scaffolding, but the architecture should support them.

## Accessibility And UX Requirements

Even though this is a game, the interface should remain:

- legible
- premium
- responsive
- accessible
- easy to navigate

Support:

- reduced motion
- readable contrast
- scalable UI where reasonable
- clear tooltips/explanations
- gradual onboarding

## Onboarding Requirements

The first-time player experience matters a lot.

The onboarding should:

- introduce the agency world
- teach the player what departments do
- introduce agent synergies
- show that failure is part of learning
- make the first operation emotionally engaging

Use the junior recruit framing here.

## Sound / Audio Direction

If audio is included, it should feel premium and restrained:

- interface cues
- ambient command-floor mood
- agent voice snippets if available
- operation tension stings

Keep the same world identity as the website/mobile project.

If full audio is too heavy for v1, architect for it cleanly.

## No Monetization

Do not build:

- pay-to-win systems
- exploitative gacha
- battle passes
- premium currency traps

If any economy is designed, it should be purely progression-oriented and player-respectful.

## Deliverable Quality Bar

The result must feel like:

- a serious game concept
- a strong playable prototype
- a coherent world
- a premium strategy product
- a natural evolution of The Agency universe

It must not feel like:

- a generic dashboard with fake stats
- a text-heavy non-game
- a shallow mission list
- a reskinned productivity app

## Implementation Phases

### Phase 1: Foundation

- import or mirror current Agency data and assets
- build game data model
- build visual system
- build shell architecture
- build save/progression architecture

### Phase 2: Core Play Loop

- HQ view
- operations board
- mission prep
- mission resolution
- basic progression

### Phase 3: Character And Story Systems

- agent progression
- trust/rivalry systems
- branching event engine
- story arcs

### Phase 4: Strategic Depth

- department modifiers
- HQ upgrades
- deeper mission variation
- replayability systems

### Phase 5: Expansion Layer

- social scaffolding
- more ops
- more world-state variation
- polish

## Final Output Requirements

At the end of the first serious build, provide:

1. a playable prototype
2. a clear explanation of the architecture
3. a list of systems implemented
4. a list of systems scaffolded for future work
5. a summary of what content is data-driven
6. known gaps
7. how to continue expanding the game

## Important Guidance

When uncertain, prefer the option that:

- preserves The Agency’s identity
- supports long-term gameplay depth
- keeps content and logic data-driven
- avoids shallow gimmicks
- makes the game more replayable
- deepens strategy and story together

Use imagination boldly, but stay grounded in the reality of what The Agency is:

an elite AI agency operating in the software-development market.

This scope is a strength, not a limitation.

Build the game so that players feel they are running and shaping the most capable AI specialist agency in the world.
